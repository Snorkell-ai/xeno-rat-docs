Parameters

Exceptions
InvalidOperationExceptionThrown when there are already too many mixer inputs.
ArgumentExceptionThrown when the input's WaveFormat does not match the mixer's WaveFormat.

This method adds a new input to the mixer. It checks if the maximum number of inputs has been reached and throws an InvalidOperationException if so. It also checks if the input's WaveFormat matches the mixer's WaveFormat and throws an ArgumentException if not.

Reads audio samples from the sources and mixes them into the provided buffer.

Parameters
bufferThe buffer to mix the audio samples into.
offsetThe offset in the buffer at which to start mixing the samples.
countThe number of samples to mix into the buffer.
Returns
The number of output samples mixed into the buffer.

This method reads audio samples from the sources, mixes them into the provided buffer starting at the specified offset, and returns the number of output samples mixed into the buffer. If the ReadFully flag is set and the output samples are less than the count, the remaining space in the buffer is filled with zeros.

◆ ReadFully

bool NAudio.Wave.SampleProviders.MixingSampleProvider.ReadFully
getset

When set to true, the Read method always returns the number of samples requested, even if there are no inputs, or if the current inputs reach their end. Setting this to true effectively makes this a never-ending sample provider, so take care if you plan to write it out to a file.
